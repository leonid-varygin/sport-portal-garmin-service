# Автоматическая переавторизация Garmin Service

## Обзор

Реализована система автоматической переавторизации для garmin-service, которая позволяет сохранять OAuth токены в файлы и восстанавливать сессии без необходимости повторного ввода пароля.

## Архитектура

### Компоненты

1. **TokenManager** (`app/services/token_manager.py`)
   - Управляет сохранением и загрузкой OAuth токенов
   - Сохраняет токены в файлы `oauth1_token.json` и `oauth2_token.json`
   - Проверяет валидность токенов и срок действия
   - Автоматически удаляет невалидные токены

2. **GarminService** (модифицированный)
   - Интегрирован с TokenManager для сохранения токенов после авторизации
   - Восстанавливает сессии из сохраненных токенов при перезапуске
   - Использует токены как первичный метод восстановления сессии

### Формат файлов токенов

#### OAuth1 Token (`user_{user_id}_oauth1_token.json`)
```json
{
    "oauth_token": "f2b49449-9318-44a1-9232-6eff96edd51a",
    "oauth_token_secret": "JqQ15zJ4G8DI8jtO9luKb1RdCpSW7KufuzY",
    "mfa_token": null,
    "mfa_expiration_timestamp": null,
    "domain": "garmin.com"
}
```

#### OAuth2 Token (`user_{user_id}_oauth2_token.json`)
```json
{
    "scope": "GARMINPAY_WRITE ATP_READ ...",
    "jti": "0c1bc45a-0342-4644-9e65-fbf0b51954f4",
    "token_type": "bearer",
    "access_token": "eyJhbGciOiJSUzI1NiIs...",
    "refresh_token": "eyJyZWZyZXNoVG9rZW5WYW...",
    "expires_in": 101955,
    "expires_at": 1765173441,
    "refresh_token_expires_in": 2591999,
    "refresh_token_expires_at": 1767663485
}
```

## Процесс работы

### 1. Первичная авторизация
1. Пользователь вводит логин/пароль
2. Система выполняет авторизацию через Garmin Connect
3. При успехе:
   - Сохраняет OAuth1 и OAuth2 токены в файлы
   - Кеширует сессию в памяти
   - Отправляет данные в основной бэкенд

### 2. Восстановление сессии (при перезапуске сервиса)
1. Проверяет наличие активной сессии в памяти
2. Если сессии нет:
   - Проверяет наличие валидных файлов токенов
   - Если токены валидны:
     - Восстанавливает токены в клиенте Garmin
     - Проверяет валидность сессии через запрос профиля
     - При успехе восстанавливает сессию без пароля
   - Если токены невалидны или отсутствуют:
     - Запрашивает данные с основного бэкенда
     - Выполняет повторную авторизацию по логину/паролю
     - Сохраняет новые токены в файлы

### 3. Автоматическая очистка
- Токены с истекшим сроком действия автоматически удаляются
- При отключении пользователя все токены удаляются
- При ошибке восстановления токены удаляются и запрашиваются заново

## API Эндпоинты

### Новые эндпоинты

#### `GET /auth/token-info/{user_id}`
Возвращает информацию о токенах пользователя:
- Наличие файлов токенов
- Валидность токенов
- Время до истечения срока действия
- Информация о активной сессии

#### Пример ответа:
```json
{
    "user_id": 1,
    "oauth1_token_exists": true,
    "oauth2_token_exists": true,
    "tokens_valid": true,
    "oauth1_file_path": "/app/tokens/user_1_oauth1_token.json",
    "oauth2_file_path": "/app/tokens/user_1_oauth2_token.json",
    "expires_in_seconds": 86345,
    "expires_in_minutes": 1439.08,
    "expires_in_hours": 23.98,
    "has_active_session": true,
    "session_created_at": "2025-12-10T01:18:33.245386",
    "session_last_used": "2025-12-10T01:18:33.245386",
    "restored_from_tokens": true
}
```

## Безопасность

### Хранение токенов
- Токены хранятся в файлах с правами доступа только для сервиса
- Файлы создаются в директории `tokens/`
- Каждому пользователю соответствуют отдельные файлы токенов
- Параметры MFA токенов сохраняются при наличии

### Валидация
- Проверка срока действия OAuth2 токена с 5-минутным буфером
- Автоматическое удаление истекших токенов
- Проверка наличия обязательных полей в токенах

## Преимущества

### Для пользователей
- **Бесшовная работа**: Нет необходимости повторно вводить пароль после перезапуска сервиса
- **Быстрое восстановление**: Сессия восстанавливается за миллисекунды
- **Надежность**: Автоматическая очистка невалидных токенов

### Для системы
- **Снижение нагрузки**: Меньше запросов к Garmin Connect для авторизации
- **Устойчивость**: Сервис продолжает работать после перезапусков
- **Масштабируемость**: Поддержка множества пользователей с индивидуальными токенами

### Требования
- Python 3.8+
- Garmin Connect библиотека
- Доступ к файловой системе для сохранения токенов

1. Убедиться, что директория `tokens/` доступна для записи
2. Настроить права доступа к файлам токенов
3. Проверить работу через API эндпоинты

## Troubleshooting

### Частые проблемы

#### Токены не сохраняются
- Проверить права доступа к директории `tokens/`
- Убедиться, что сервис запущен с нужными привилегиями

#### Восстановление не работает
- Проверить валидность файлов токенов
- Убедиться, что `expires_at` не истек
- Проверить логи на предмет ошибок

#### Сессия отваливается
- Проверить срок действия refresh токена
- Убедиться, что сервис имеет доступ к файлам токенов
- Проверить сетевое подключение к Garmin
